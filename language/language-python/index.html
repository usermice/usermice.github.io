    1. Python编码规范
  • 6.1 代码缩进
  • 6.1.1 在Python中，代码缩进推荐使用四个空格，而不是一个tab键
  • 6.2 最大行宽
  • 6.2.1 限制所有行的最大行宽为 79 字符。文本长块，比如文档字符串或注释，行长度应限制为 72 个字符
  • 6.3 空行
  • 6.3.1 使用空行分割顶层函数和类的定义。类中的方法的定义使用单个空行进行分割
  • 6.3.2 额外的空行可以必要的时候用于分割不同的函数组，但是要尽量节约使用
  • 6.3.3 额外的空行可以必要的时候在函数中用于分割不同的逻辑块，但是要尽量节约使用
  • 6.4 空格
  • 6.4.1 各种右括号前不要加空格
  • 6.4.2 逗号、冒号、分号前不要加空格
  • 6.4.3 函数的左括号前不要加空格，如 Func(1)
  • 6.4.4 序列的左括号前不要加空格，如 list[2]
  • 6.4.5 操作符左右各加一个空格，不要为了对齐增加空格
  • 6.4.6 函数默认参数使用的赋值符左右省略空格
  • 6.5 import导入
  • 6.5.1 import导入始终在文件的顶部，在模块注释和文档字符串之后，在模块全局变量和常量之前
  • 6.5.2 导入顺序如下：标准库进口，相关的第三方库，本地库。各组的导入之间要有空行
  • 6.5.3 不要在一行import多个库，每行只能import一个库
  • 6.6 注释
  • 6.6.1 总体原则，错误的注释不如没有注释。所以当一段代码发生变化时，第一件事就是要修改注释
  • 6.6.2 注释最好使用英文，最好是完整的句子，首字母大写，句后要有结束符，结束符后跟两个空格，开始下一句。如果是短语，可以省略结束符
  • 6.6.3 块注释，在一段代码前增加的注释。行注释，在一句代码后加注释。但是这种方式尽量少使用
  • 6.7 命名规则
  • 6.7.1 模块命名尽量短小，使用全部小写的方式，可以使用下划线
  • 6.7.2 包命名尽量短小，使用全部小写的方式，不可以使用下划线
  • 6.7.3 类的命名使用 CapWords 的方式，模块内部使用的类采用 _CapWords 的方式
  • 6.7.4 异常命名使用 CapWords+Error 后缀的方式
  • 6.7.5 全局变量尽量只在模块内有效，类似C语言 中的static
  • 6.7.6 函数命名使用全部小写的方式，可以使用下划线
  • 6.7.7 常量命名使用全部大写的方式，可以使用下划线
  • 6.7.8 类的属性（方法和变量）命名使用全部小写的方式，可以使用下划线。类的属性若与关键字名字冲突，后缀一下划线，尽量不要使用缩略等其他方式
  • 6.7.9 为避免与子类属性命名冲突，在类的一些属性前，前缀两条下划线
  • 6.7.10 类的方法第一个参数必须是self，而静态方法第一个参数必须是cls

方法作用
uuid.uuid1()基于MAC地址，时间戳，随机数来生成唯一的uuid，可以保证全球范围内的唯一性
uuid.uuid2()算法与uuid1不同，不同的是把时间戳的前四位置换位POSI的UID，不过需要注意的是python中没有基于DCE的算法，所以python的uuid模块中没有uuid2这个方法
uuid.uuid3(namespace，name)通过计算一个命名空间和名字的MD5散列值来给出一个uuid，所以可以保证命名空间中的不同名字具有不同的uuid，但是相同的名字就是相同的uuid，namesp并不是一个自己手动指定的字符串或其他量，而是在uuid模块中本身给出的一些值。比如：uuid.NAMESPACE_DNS，uuid.NAMESPACE_URL，uuid.NAMESPACE_OID等，这些值也是uuid对象，根据一定的规则计算得出
uuid.uuid4()通过伪随机数得到uuid，有一定重复的概率性，
uuid.uuid5(namespace，name)和uuid3基本相同，只不过采用的散列算法是sha1
