没有Rancher Server情况下管理下游Kubernetes集群

1. Rancher服务宕机情况下使用Kubernetes Cluster

  • Rancher Server是上游中心管理角色，控制或创建旗下N个Kubernetes Cluster集群，不管是测试中的单个节点或者是Kubernetes Cluster集群失效都无法读取，运维仍需要对下游所有Kubernetes Cluster集群管控
服务器 IP CPU 内存
master 192.168.35.128 8core 8G
node-1 192.168.35.129 4core 4G
node-2 192.168.35.130 4core 4G
  • 前提条件
    所有机器都是CentOS 7
    Rancher Server是以Docker方式单独建立
    test-cluster Kubernetes Cluster集群是以Rancher Server自定义方式创建
  • 模拟Racnher Server服务失效对Kubernetes Cluster集群进行管控
    • 点击创建集群名称

  • 提前下载Kubernetesfig文件

2. 模拟Rancher服务宕机

  • 停止Rancher Server服务docker stop rancher
    • 查看Kubernetes集群版本号

2.1 安装Kubectl

  • 在能连接test-kubernetes-cluster集群中任意一个节点部署Kubectl
  • Kubectl版本需和test-cluster-cluster版本相同或者高于

2.2 创建.kube目录修改文件

  • 并且将下载的Kubernetesfig文件改名为config，然后放到${HOME}/.kube/目录中
  • ${HOME}/.kube/config文件没修改前
  • ${HOME}/.kube/config文件修改后
