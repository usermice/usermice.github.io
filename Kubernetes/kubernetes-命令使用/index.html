3. kubectl资源、命令

  • kubectl：是kubernetes集群的命令行工具，通过它能够对集群本身进行管理，并能够在集群上进行容器化应用的安装和部署

3.1 kubectl api-resources：是查看kubernetes资源命令，Kubernetes中所有的内容都抽象为资源，所有命令都是围绕资源操作、执行。

资源分类 资源名称 缩写 资源作用
集群级别资源 nodes no 集群组成部分
集群级别资源 namespaces ns 隔离Pod
Pods Pods po 装载容器
Pod资源控制器 replicationcontrollers rc 比较原始的Pod控制器，已经被废弃，由ReplicaSet替代。
— replicasets rs 保证指定数量的Pod运行，并支持Pod数量变更，镜像版本变更
— deployments deploy 通过控制ReplicaSet来控制Pod，并支持滚动升级、版本回退。
— daemonsets ds 在集群中的指定Node上都运行一个副本，一般用于守护进程类的任务。
— jobs — 它创建出来的Pod只要完成任务就立即退出，用于执行一次性任务。
— cronjobs cj 它创建的Pod会周期性的执行，用于执行周期性的任务。
— horizontalpodautoscalers hpa 可以根据集群负载自动调整Pod的数量，实现削峰填谷。
Pod资源控制器 statefulsets sts 管理有状态的应用。
服务发现资源 services svc 统一Pod对外接口
服务发现资源 ingress ing 统一Pod对外接口
存储资源 volumeattachments — 存储
— persistentvolumes pv 存储
存储资源 persistentvolumeclaims pvc 存储
配置资源 configmaps cm 配置
配置资源 secrets — 配置

3.2 基本命令详解

  • kubectl --help：可以通过--help查看详细的操作命令
基本命令(初学者) 命令 翻译 命令作用
— create 创建 从文件或标准输入创建资源
— expose 暴露 将复制控制器、服务、部署或pod暴露为新的Kubernetes服务Service
— run 运行 在集群中运行一个指定的镜像
基本命令(初学者) set 设置 设置一个对象的特定特征
  • kubectl命令的语法如下$ kubectl [command] [type] [name] [flags]
    • command: 指定要对资源执行的操作，比如：create、get、delete。
    • type：指定资源的类型，比如：deployment、pod、service。
    • name： 指定资源的名称，名称大小写敏感
    • flags： 指定额外的可选参数
3.2.1 kubectl create --help：创建命令用于创建资源
  • yaml文件详解、使用kubectl create命令生成yaml文件
    • deployment：无状态应用部署
    • java-demo：自定义名称
    • --image：使用镜像
    • --dry-run：测试
    • -o：输出
    • >/tmp/deploy.yaml：定向输出到文件，不加>/tmp/deploy.yaml 输出到屏幕查看
3.2.2 kubectl expose --help端口暴露
  • expose 端口设置固定选项
  • java-demo： 自定义名称
  • --port：Pod的Service服务端口，集群内部互通端口
  • --target-port：Pod容器内服务端口端口
  • --type：随机生成外部访问端口
  • >java-demo-svc.yaml：定向输出到文件，不加>java-demo-svc.yaml输出到屏幕查看
3.2.3 kubectl run --help：命令使用镜像创建容器
3.2.4 kubectl set --help更新镜像、设置一个对象的特定特征

3.3 中级基本命令：运行、调试命令

基本命令(中级) 命令 翻译 命令作用
— explain 解释 展示资源类型、属性文档
— get 获取 获取显示一个或多个资源
— edit 编辑 编辑服务器上的资源
基本命令(中级) delete 删除 delete 按文件名、标准输入、资源和名称，或按资源和标签选择器删除资源
3.3.1 kubectl explain --help：展示资源文档
3.3.2 kubectl get --help：获取资源
3.3.3 kubectl edit --help：cm：编辑configMap中的配置
3.3.4 kubectl delete --help：删除

3.4 Kubernetes部署命令

部署命令 命令 翻译 命令作用
— rollout 首次展示 管理资源的发布
— scale 规模 扩(缩)容Pod的数量,为Deployment、ReplicaSet或Replication Controller设置新的大小
— autoscale 自动调整 自动调整Pod的数量，自动缩放Deployment、ReplicaSet、StatefulSet或ReplicationController
3.4.1 kubectl rollout --help：管理资源
    1. status：显示当前升级状态
    1. history：显示升级历史记录
    1. pause：暂停版本升级过程
    1. resume：继续已经暂停的版本升级过程
    1. restart：重启版本升级过程
    1. undo：回滚到上一级版本 （可以使用--to-revision，指定版本）
    1. --record：记录CHANGE-CAUSE(改变原因)操作记录,便于回滚
3.4.2 kubectl scale --help：扩缩容Pod的数量
  • --replicas：参数定义Pod数量
3.4.3 kubectl autoscale --help：使Deployment设定，使用默认的自动伸缩策略，指定目标CPU使用率，使其Pod数量在2到10之间

3.5 集群管理命令

集群管理命令 命令 翻译 命令作用
— certificate 证书 修改证书资源
— cluster-info 集群信息 显示集群信息
— top 顶部显示资源（CPU/内存）使用情况
— cordon 警戒线 警戒线 将节点标记为不可调度
— uncordon uncordon将节点标记为可调度
— drain 排水 排水节点准备维护
集群管理命令 taint 污点 taint更新一个或多个节点上的污点
3.5.3 kubectl top --help：资源使用情况
  • metrics-servermetrics-server可以用来收集集群中的资源使用情况、需要单独安装
  • Kubernetes指标服务器
  • hostNetwork: true：新增参数
  • image: registry.cn-hangzhou.aliyuncs.com/google_containers/metrics-server-amd64:v0.3.6 args:：国内修改为阿里云地址
  • - --kubelet-insecure-tls：新增参数
  • - --kubelet-preferred-address-types=InternalIP,Hostname,InternalDNS,ExternalDNS,ExternalIP：替换参数
3.5.7 kubectl taint --help：污点配置

3.6 故障排除和调试命令

故障排除和调试命令 命令 翻译 命令作用
— describe 描述 describe显示特定资源或资源组的详细信息
— logs 日志 打印 pod 中容器的日志
— attach 缠绕 进入运行中的容器、附加到正在运行的容器
— exec 执行 在容器中执行命令
— port-forward 转发 将一个或多个本地端口转发到一个pod
— proxy 代理 运行到Kubernetes API服务器的代理
— cp 复制 在Pod内外复制文件
— auth 检查授权
故障排除和调试命令 debug 调试 创建调试会话以对工作负载和节点进行故障排除
3.6.1 kubectl describe --help显示资源内部信息
3.6.2 kubectl logs --help日志查看
3.6.4 kubectl exec --help进入容器
3.6.5 kubectl port-forward --help转发Pod中端口

3.7 高级命令

高级命令 命令 翻译 命令作用
— diff 对比 对比实时版本与将要应用的版本之间的差异
— apply 应用 通过文件名或标准输入将配置应用于资源
— patch 更新 更新一个资源，更新资源的字段
— replace 替换 用文件名或标准输入替换资源
— wait 等待 等待实验：等待一个或多个资源的特定条件
高级命令 kustomize 从目录或 URL 构建一个 kustomization 目标
3.7.2 kubectl apply --help
  • 如果资源不存在，就创建，相当于kubectl create。
  • 如果资源存在，就更新，相当于kubectl patch。
3.7.3 kubectl patch --help：补丁修改、更新资源的字段
3.7.4 kubectl replace --help：根据yaml文件更新修改后配置资源，会停掉原来的资源，重新创建

3.8 设置命令

— completion 完成输出指定shell的shell完成代码（bash或zsh）
