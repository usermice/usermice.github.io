3. kubectl命令

  • kubectl是kubernetes集群的命令行工具，通过它能够对集群本身进行管理，并能够在集群上进行容器化应用的安装和部署
  • kubectl命令的语法如下$ kubectl [command] [type] [name] [flags]
    • command: 指定要对资源执行的操作，比如：create、get、delete。
    • type：指定资源的类型，比如：deployment、pod、service。
    • 可以通过–help查看详细的操作命令：kubectl --help
    • kubernetes 中所有的内容都抽象为资源，可以通过kubectl api-resources进行查看

3.1.1 基本命令详解

— 命令 翻译 命令作用
基本命令 create 创建 创建一个资源
— edit 编辑 编辑一个资源
— get 获取 获取一个资源
— patch 更新 更新一个资源
— delete 删除 删除一个资源
基本命令 explain 解释 展示资源文档
  • yaml文件详解、yaml文件生成 、create创建命令
  • create：创建
  • deployment：无状态应用部署
  • java-demo：自定义名称
  • --image：使用镜像
  • --dry-run：测试
  • -o：输出
  • > deploy.yaml：定向输出到文件，不加>deploy.yaml 输出到屏幕查看

3.1.2 kubectl create：创建命令用于创建资源

  • name：指定资源的名称，名称大小写敏感
  • flags：指定额外的可选参数
  • 使用kubectl create命令生成yaml文件

3.1.3 kubectl edit：cm：编辑configMap中的配置

3.1.4 kubectl get：获取资源

3.1.5 kubectl patch：补丁修改、更新资源的字段

3.1.6 kubectl delete：删除

3.1.7 kubectl explain：展示资源文档

3.2.1 运行、调试命令

— 命令 翻译 命令作用
运行和调试 run 运行 在集群中运行一个指定的镜像
— expose 暴露 暴露资源为Service
— describe 描述 显示资源内部信息
— logs 日志 输出容器在Pod中的日志
— attach 缠绕 进入运行中的容器
— exec 执行 执行容器中的一个命令
— cp 复制 在Pod内外复制文件
— rollout 首次展示 管理资源的发布
— scale 规模 扩（缩）容Pod的数量
运行和调试 autoscale 自动调整 自动调整Pod的数量
高级命令 apply 应用 通过文件对资源进行配置
高级命令 label 标签 更新资源上的标签
其他命令 cluster-info 集群信息 显示集群信息
其他命令 version 版本 输出 client 和 server 的版本信息

3.2.2 kubectl run：命令使用镜像创建容器

3.2.3 kubectl expose端口暴露

  • expose 端口设置固定选项
  • java-demo： 自定义名称
  • --port：集群内部互通端口
  • --target-port：容器内部端口
  • --type：随机生成外部访问端口
  • >java-demo-svc.yaml：定向输出到文件，不加>java-demo-svc.yaml输出到屏幕查看

3.2.4 kubectl describe显示资源内部信息

3.2.5 kubectl logs日志查看

3.2.6 kubectl exec进入容器

3.2.7 kubectl rollout、set升级、undo回滚、

  • status：显示当前升级状态
  • history：显示升级历史记录
  • pause：暂停版本升级过程
  • resume：继续已经暂停的版本升级过程
  • restart：重启版本升级过程
  • undo：回滚到上一级版本 （可以使用--to-revision，指定版本）

3.2.8 kubectl scale：扩缩容Pod的数量

3.2.9 kubectl autoscale：使Deployment设定，使用默认的自动伸缩策略，指定目标CPU使用率，使其Pod数量在2到10之间

3.2.10 kubectl apply

  • 如果资源不存在，就创建，相当于kubectl create。
  • 如果资源存在，就更新，相当于kubectl patch。
